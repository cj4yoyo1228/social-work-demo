# 社工管理系統 — 三個互動原型設計

> 日期：2026-02-22
> 狀態：已核准

## 專案概要

為社工管理系統企劃書產出三個可互動 HTML 原型，部署到線上網址供評審操作體驗。

## 技術方案

- **方案**：純 HTML + Tailwind CDN + 原生 JavaScript
- **部署**：Vercel 靜態託管（免費）
- **互動深度**：基本互動（狀態切換、標籤選取、數字動態更新），資料存在記憶體中

## Demo 1：班級管理中心（手機版）

### 目標受眾

大學生社工

### 設計核心

極簡、零學習成本、一鍵完成例行公事

### 畫面流程（3 個狀態）

**狀態 1：今日任務列表**

- 問候語 + 今日班級數量
- 每個班級一張卡片：班名、時間、學生數、「開始點名」按鈕

**狀態 2：點名畫面**

- 頂部：班名 + 即時出席人數
- 學生列表：預設全勤（綠色勾勾）
- 點擊學生 → 切換為缺席/請假，出現下拉選單和原因輸入框
- 底部：「完成點名」大按鈕

**狀態 3：點名完成 + 日誌入口**

- 完成標記 + 出席統計摘要
- 缺席學生列表（附原因）
- 「撰寫今日日誌」大按鈕
- 返回首頁按鈕

### 互動細節

- 預設全勤、例外管理邏輯
- 出席人數即時更新
- 狀態間平滑切換

### 裝置模擬

固定寬度 390px，居中顯示，加手機外框

## Demo 2：個案服務紀錄（手機/電腦通用）

### 目標受眾

大學生社工、各校督導

### 設計核心

結構化輸入、降低打字負擔

### 畫面流程（3 個狀態）

**狀態 1：學生列表**

- 搜尋框
- 學生卡片：姓名、班級、標籤、最近紀錄日期

**狀態 2：學生個別頁面**

- 基本資訊區塊
- 近期紀錄時間軸
- 「新增紀錄」按鈕

**狀態 3：新增紀錄（結構化輸入）**

- 觀察面向標籤（可多選）：課堂參與、人際互動、情緒、學習表現、家庭狀況
- 簡短描述文字框
- Phase 2 預留：語音輸入 icon（灰色不可點）、AI 輔助整理按鈕（灰色不可點）
- 儲存後回到學生頁面

### 互動細節

- 標籤多選切換
- 文字輸入
- 儲存後新紀錄出現在列表最上方
- Phase 2 按鈕 hover 提示「即將推出」

### 裝置模擬

響應式設計，手機和電腦都能用

## Demo 3：行政管理報表（電腦版 Dashboard）

### 目標受眾

總部、分校、各校督導

### 設計核心

一目了然的異常管理、省去人工催收

### 畫面配置（單頁）

**區塊 1：KPI 卡片列**

- 班級總數（12）
- 今日出席率（94.2%，✅ 正常）
- 日誌繳交率（78.5%，⚠️ 偏低）
- 需關注項目（3，🔴 注意）

**區塊 2：異常預警列表**

- 連續未交日誌的社工（附「一鍵提醒」）
- 請假率異常的班級（附「查看詳情」）
- 尚未點名的社工（附「一鍵提醒」）

**區塊 3：出席趨勢圖**

- CSS/Canvas 折線圖，顯示本週一到五趨勢

**區塊 4：班級狀態總覽表格**

- 欄位：班級、社工、出席率、日誌狀態、狀態
- 可按出席率排序

### 互動細節

- KPI 數字進場動畫（從 0 跳到最終值）
- 「一鍵提醒」→ 變成「✅ 已提醒」
- 「查看詳情」→ 展開明細
- 表格可排序

### 裝置模擬

全寬桌面版面
